# 区间重叠策略详解

## 问题背景

在分区域处理大规模GVCF文件时，面临两个挑战：

1. **边界变异丢失**：如果区间不重叠，边界处的变异可能因位置判断问题而被遗漏
2. **重复记录**：如果区间简单重叠，边界变异会在多个区间重复出现

## 解决方案：双重坐标系统

采用**提取区间（带重叠）**和**过滤区间（不重叠）**的双重坐标系统。

### 示例（chr1H染色体，5Mb窗口，20kb重叠）

```
位置 (bp)       1      4,990,000  5,000,000  5,180,001  5,190,000  10,000,000
                |           |          |          |           |           |
                ├───────────┴──────────┤          ├───────────┴───────────┤
区间1           │  提取区间 (1-5,000,000)         │
                │                                 │
                ├──过滤区间 (1-4,990,000)────────┤
                                                 │
区间2                                           │  提取区间 (4,980,001-10,000,000)
                                                │  (与区间1重叠20kb)
                                                 │
                                                 ├──过滤区间 (4,990,001-9,990,000)─┤
```

### 详细对比

| 区间  | 提取区间 (带重叠)    | 过滤区间 (不重叠)    | 重叠部分  |
| ----- | -------------------- | -------------------- | --------- |
| 区间1 | 1-5,000,000          | 1-4,990,000          | -         |
| 区间2 | 4,980,001-10,000,000 | 4,990,001-9,990,000  | 20,000 bp |
| 区间3 | 9,980,001-15,000,000 | 9,990,001-14,990,000 | 20,000 bp |

### 工作流程

#### Step 1: 提取阶段（使用重叠区间）

```bash
# 区间1: 提取完整窗口
bcftools view -r chr1H:1-5000000 sample.gvcf.gz > interval1.gvcf

# 区间2: 提取带重叠的窗口
bcftools view -r chr1H:4980001-10000000 sample.gvcf.gz > interval2.gvcf
```

**结果：**

- 位置 4,980,001-5,000,000 的变异在两个区间都被提取
- 确保边界变异不会丢失

#### Step 2: 合并阶段（使用非重叠过滤区间）

```bash
# 从区间1提取非重叠部分
zcat interval1.snp.vcf.gz | awk '$2>=1 && $2<=4990000' > chr1H.part1.vcf

# 从区间2提取非重叠部分
zcat interval2.snp.vcf.gz | awk '$2>=4990001 && $2<=9990000' > chr1H.part2.vcf

# 合并
cat chr1H.part1.vcf chr1H.part2.vcf | bgzip > chr1H.snp.vcf.gz
```

**结果：**

- 每个位置只出现一次
- 没有重复记录

## 数值计算公式

假设：

- 窗口大小 = W
- 重叠大小 = O
- 当前窗口序号 = n (从0开始)

### 第一个窗口 (n=0)

```
提取区间: 1 到 W
过滤区间: 1 到 (W - O)
```

### 后续窗口 (n > 0)

```
提取区间起始: n × (W - O) + 1
提取区间结束: min((n+1) × (W - O) + O, 染色体长度)

过滤区间起始: n × (W - O) + 1
过滤区间结束: min((n+1) × (W - O), 染色体长度)
```

### 实际计算示例 (W=5Mb, O=20kb)

```
区间0 (n=0):
  提取: 1 - 5,000,000
  过滤: 1 - 4,990,000

区间1 (n=1):
  提取: 1 × (5,000,000 - 20,000) + 1 = 4,980,001
        到 5,000,000 - 20,000 + 5,000,000 = 9,980,000 + 20,000 = 10,000,000
  提取: 4,980,001 - 10,000,000

  过滤: 4,980,001
        到 2 × (5,000,000 - 20,000) = 9,960,000
  过滤: 4,990,001 - 9,990,000

区间2 (n=2):
  提取: 9,980,001 - 15,000,000
  过滤: 9,990,001 - 14,990,000
```

## 参数选择建议

### 窗口大小 (W)

| 染色体大小 | 建议窗口 | 区间数量 | 说明       |
| ---------- | -------- | -------- | ---------- |
| < 50 Mb    | 5 Mb     | < 10     | 小染色体   |
| 50-200 Mb  | 5-10 Mb  | 10-40    | 中等染色体 |
| > 200 Mb   | 10-20 Mb | 20-40    | 大染色体   |

### 重叠大小 (O)

| 变异密度 | 建议重叠  | 说明          |
| -------- | --------- | ------------- |
| 低       | 10,000 bp | SNP较少       |
| 中等     | 20,000 bp | 一般情况      |
| 高       | 50,000 bp | 高密度SNP区域 |

**计算原则：**

- 重叠至少应覆盖最长的INDEL长度（通常<10kb）
- 建议 20,000 bp (20kb) 作为默认值
- 重叠过大会增加计算量，过小可能遗漏边界变异

## 验证方法

### 1. 检查区间文件

```bash
# 查看区间生成结果
cat intervals.txt | head

# 统计区间数量
wc -l intervals.txt

# 检查重叠
awk 'NR>1 {
    if ($2 <= prev_end) {
        overlap = prev_end - $2 + 1
        print "Overlap:", $1, $2, prev_end, overlap
    }
    prev_end = $3
}' intervals.txt
```

### 2. 验证边界变异

```bash
# 检查区间1的结尾部分
zcat interval_1/sort.snp.vcf.gz | awk '$2>=4980000 && $2<=5000000' | wc -l

# 检查区间2的开始部分
zcat interval_2/sort.snp.vcf.gz | awk '$2>=4980000 && $2<=5000000' | wc -l

# 两者应该都有记录（重叠区域）
```

### 3. 验证最终VCF无重复

```bash
# 检查边界位置是否只出现一次
zcat chr1H.snp.vcf.gz | awk '$2>=4990000 && $2<=4990100' | wc -l

# 应该约等于该区间的变异密度
```

## 常见问题

### Q1: 为什么需要重叠？

**A:** 避免边界变异丢失。某些变异检测工具在区间边界处的判断可能不准确，重叠可以确保边界变异被完整捕获。

### Q2: 重叠大小如何选择？

**A:** 建议默认 20kb。这个值足以覆盖大多数INDEL和检测工具的边界不确定性，同时不会显著增加计算负担。

### Q3: 如果染色体长度不是窗口大小的整数倍怎么办？

**A:** 最后一个窗口会自动调整大小，确保覆盖到染色体末端。

### Q4: 为什么要区分提取区间和过滤区间？

**A:**

- 提取区间重叠：确保完整性
- 过滤区间不重叠：避免重复记录

### Q5: 可以跳过重叠策略吗？

**A:** 技术上可以（设置 overlap=0），但不推荐。可能导致边界变异丢失，特别是高密度变异区域。

## 性能影响

| 重叠大小  | 额外计算量 | 边界安全性 | 推荐场景   |
| --------- | ---------- | ---------- | ---------- |
| 0 bp      | 0%         | 低         | 测试用途   |
| 10,000 bp | ~2%        | 中         | 低密度变异 |
| 20,000 bp | ~4%        | 高         | 一般情况   |
| 50,000 bp | ~10%       | 很高       | 高密度变异 |

**结论：** 20kb重叠是计算效率和安全性的最佳平衡点。
